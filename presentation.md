# 🛡️ Fake Engagement Detection: Project Presentation & Documentation

Below is the comprehensive architectural and systemic breakdown of the Fake Engagement Detection project, designed to be adapted for pitch decks, read-me files, or live presentations to judges.

---

## 1. Problem Statement
Social media platforms increasingly suffer from artificially manufactured engagement generated by bot farms or coordinated actor rings. This distorts genuine behavioral patterns, misleads algorithms, and damages organizational/brand trust. 

**The Challenge:** Build a behavioral detection model that operates in real-time, differentiates organic versus artificial engagement, detects coordinated anomalies, and provides actionable, human-readable explanations for its findings without sacrificing performance.

---

## 2. Data & Features
To test and validate the engine, we utilized the rigorous **TwiBot-22** dataset (highly imbalanced human/bot interaction labeled data). In production, the system securely integrates with the **YouTube Data API v3** to pull live, top-level comments sequentially.

**Key Behavioral Features Extracted:**
- **Linguistic Complexity:** Semantic spacing, generic praise detection, repetitive word patterns.
- **Timing Regularity (`1min` buckets):** Mathematical intervals calculated via standard deviations to flag suspiciously exact, machine-like posting frequencies.
- **Engagement Burst Metrics:** Time-series analysis to find statistically improbable spikes in traffic indicative of a bot swarm.
- **Relational Interaction (Graph Mapping):** Multi-hop cross-account comment analysis.

---

## 3. Model
We upgraded from standard keyword-frequency models (TF-IDF/Random Forest) to a highly capable **Deep Learning Semantic Architecture**:
* **LLM Embedder Layer (`all-MiniLM-L6-v2`):** A HuggingFace transformer that immediately encodes raw text into 384-dimensional dense semantic vectors. This allows the system to comprehend *context, sarcasm, and sentence meaning* rather than just parsing raw words.
* **Classification Layer (MLP Neural Network):** Operating on the dense vectors, a highly tuned Multi-Layer Perceptron (128x64 layer density) predicts the binary string (Human vs. Bot). 
* **Explainable AI (XAI):** Utilizing `lime.lime_text`, the system generates "Local Interpretability" reports, rendering a beautifully formatted breakdown of exactly which words positively or negatively influenced the Neural Network.

---

## 4. Architecture
The architecture comprises four distinct engines functioning behind a modern, dark-themed Streamlit UI.

1. **Ingestion Engine:** Live YouTube API calls chunked efficiently.
2. **AI Semantic Engine:** SentenceTransformers + MLP inference.
3. **Graph Database Engine:** Converts users and comments into Nodes and Edges asynchronously loaded into a local **Neo4j** instance. Utilizing complex `Cypher` pattern queries, it finds multi-user bot rings (e.g. 5+ distinct accounts commenting within the same 60 seconds).
4. **Scoring & Fusion Engine:** Consolidates Graph Risk, Timing Risk, and NLP signals into a weighted, global **Authenticity Assessment Score** (0-100%).
5. **Visualization Layer:** Outputs an organic, physics-based 3D Network diagram natively in the browser via `pyvis`.

---

## 5. Results
- **Model Accuracy:** The LLM-Embedding + Neural Network achieved a **91% overall accuracy** and an impressive **0.92 F1-score** for bot detection on the Kaggle TwiBot-22 validation set.
- **Scale:** By leveraging lightweight transformers mapping to the CPU locally, the dashboard analyzes 500 comments, constructs a Neo4j database, and outputs XAI explanations in seconds.
- **Unified UI:** The Streamlit application effectively consolidates 3D graphing, lime explainability, timeline metrics, and scoring gauges into a single, cohesive command center.

---

## 6. Behavioural Insights
A key differentiator is the system's ability to translate raw algorithmic flags into human-readable *Anomaly Explanations*:
* *"Posted at exact machine-like intervals"*
* *"Identified in a coordinated posting ring"*
* *"Contains highly repetitive or non-contextual phrasing"*

By surfacing *why* an account is flagged instead of just *that* it was flagged, the platform moves beyond simple classification to true, actionable cybersecurity intelligence.

---

## 7. Practical Impact
1. **Platform Integrity:** Immediate value to platforms looking to purge bot rings natively during high-tension events (elections, product launches).
2. **Audit & Compliance:** The integration of Local Interpretability (XAI) and Neo4j Graph snapshots means all automated moderation actions maintain a transparent, reviewable "paper trail."
3. **Brand & Influencer Vetting:** Marketing teams can instantly plug in a creator's content and objectively measure if their ROI is being falsely inflated by comment-farming vendors.

---

## 8. Future Scope
* **Multi-Platform Ingestion:** Expanding beyond YouTube to Reddit, TikTok, or Instagram live-streams using the exact same underlying embedding/graph architecture.
* **Continuous Online Learning:** Enabling active-learning loops where moderators can "Accept/Reject" the XAI explanations, feeding that data dynamically back into the MLP Neural Network.
* **Deep Graph Neural Networks (GNN):** Translating the Neo4j sub-graph relationships directly into node feature inputs—fusing text embeddings and graph structure into one holistic GNN layer.
